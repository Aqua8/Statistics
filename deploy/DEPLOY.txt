========================================

항목              로컬                    배포
-------------------------------------------------
COOKIE_SECURE     false                   true
DB 포트 노출      있음 (${DB_PORT})       없음
HTTPS             없음 (HTTP만)           Let's Encrypt SSL
포트              80                      80, 443
restart 정책      없음                    unless-stopped
certbot           없음                    자동 갱신 (12시간마다)
nginx 설정        proxy/nginx.conf        deploy/nginx/nginx.conf.template
Portfolio 빌드    ../Portfolio            ../../Portfolio


========================================
VM 최초 배포 순서
========================================

1. Docker 설치 (Debian 기준 — Ubuntu와 다름)

   # 공식 저장소 추가
   sudo apt update && sudo apt install -y ca-certificates curl gnupg
   sudo install -m 0755 -d /etc/apt/keyrings
   curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
   echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
   sudo apt update

   # 설치
   sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
   sudo usermod -aG docker $USER
   newgrp docker

2. SSH 키 생성 및 GitHub 계정 등록

   ssh-keygen -t ed25519 -C "vm-deploy" -f ~/.ssh/id_ed25519 -N ""
   cat ~/.ssh/id_ed25519.pub  # 이 값을 GitHub에 등록

   GitHub → Settings → SSH and GPG keys → New SSH key
   ※ 레포별 deploy key 아닌 계정 SSH key로 등록해야 모든 레포 접근 가능

   ssh -T git@github.com  # 연결 확인

3. 레포 클론 (같은 레벨에 위치해야 함)

   git clone git@github.com:Aqua8/Statistics.git
   git clone git@github.com:Aqua8/Portfolio.git
   git clone git@github.com:Aqua8/MatJip.git

   디렉토리 구조:
   ~/
   ├── Statistics/
   ├── Portfolio/
   └── MatJip/

   ※ MatJip 백엔드는 prod 프로파일(application-prod.yml, 커밋됨)을 사용하며
     스키마는 기동 시 schema.sql(CREATE TABLE IF NOT EXISTS)로 자동 생성됨 → 별도 init.sql 불필요
   ※ MatJip 이미지 업로드(GCS)는 VM 서비스 계정의 ADC를 사용하므로
     VM 서비스 계정에 해당 GCS 버킷 권한(objectAdmin)이 있어야 함. 키 파일 마운트 불필요.

4. 도메인 DNS 설정
   - Duck DNS (duckdns.org) 또는 구매한 도메인 사용
   - STATS_DOMAIN → VM 외부 IP
   - PORTFOLIO_DOMAIN → VM 외부 IP
   - MATJIP_DOMAIN → VM 외부 IP
   - 전파까지 수 분 ~ 수십 분 소요

5. GCP 방화벽 규칙 추가
   GCP 콘솔 → VPC 네트워크 → 방화벽 → 방화벽 규칙 만들기
   - 트래픽 방향: 수신 / 소스: 0.0.0.0/0 / 프로토콜: TCP 80, 443

6. .env 파일 작성

   cd ~/Statistics/deploy
   cp .env.example .env
   nano .env  # 값 채우기

   ※ JWT_SECRET 생성: openssl rand -base64 32

7. init.sql 업로드
   ※ init.sql은 .gitignore에 포함되어 레포에 없음 → 로컬에서 직접 복사 필요
   로컬에서 실행:
   scp ~/Desktop/Development/Statistics/init.sql <user>@<vm-ip>:~/Statistics/

8. SSL 초기화 및 서비스 시작 (최초 1회)

   ./scripts/init-ssl.sh

   ※ SSH 세션이 끊길 수 있으므로 tmux 또는 screen 사용 권장
   tmux new -s deploy
   ./scripts/init-ssl.sh
   # 끊겨도 tmux attach -t deploy 로 재접속

9. 게스트 유저 / 프로젝트 DB 데이터 삽입
   ※ init.sql에 포함되어 있지 않은 경우 수동 삽입 필요

   set -a; source .env; set +a

   docker compose exec db mariadb -uroot -p$DB_ROOT_PASSWORD $DB_NAME -e "
   INSERT INTO users (id, email, password, name, created_at, del_yn)
   VALUES (<GUEST_USER_ID>, '<email>', '<bcrypt_password>', '<name>', NOW(), 'N');

   INSERT INTO projects (id, user_id, name, domain, tracking_key, created_at, del_yn)
   VALUES (<GUEST_PROJECT_ID>, <GUEST_USER_ID>, '<name>', '<domain>', '<tracking_key>', NOW(), 'N');
   "

10. 정상 동작 확인

    docker compose ps
    docker compose logs proxy


========================================
DB 통합 마이그레이션 (matjip-db → db, 기존 서버 전용)
========================================

단일 MariaDB 컨테이너(db)에 matjip 스키마를 통합하는 작업.
※ 새 서버 배포 시에는 이 절차 불필요 (init 스크립트가 자동 처리)

1. matjip-db 데이터 백업

   set -a; source .env; set +a
   docker compose exec matjip-db mariadb-dump \
     -u root -p$MATJIP_DB_ROOT_PASSWORD \
     --single-transaction $MATJIP_DB_NAME > matjip_backup.sql

2. matjip-backend 정지 (데이터 변경 차단)

   docker compose stop matjip-backend

3. .env에서 MATJIP_DB_ROOT_PASSWORD 줄 제거

4. 코드 업데이트

   git pull origin main

5. 통합 db에 matjip 스키마·유저 생성

   set -a; source .env; set +a
   docker compose exec db mariadb -u root -p$DB_ROOT_PASSWORD -e "
     CREATE DATABASE IF NOT EXISTS \`$MATJIP_DB_NAME\`;
     CREATE USER IF NOT EXISTS '$MATJIP_DB_USERNAME'@'%' IDENTIFIED BY '$MATJIP_DB_PASSWORD';
     GRANT ALL ON \`$MATJIP_DB_NAME\`.* TO '$MATJIP_DB_USERNAME'@'%';
     FLUSH PRIVILEGES;
   "

6. 백업 데이터 복원

   docker compose exec -T db mariadb \
     -u root -p$DB_ROOT_PASSWORD $MATJIP_DB_NAME < matjip_backup.sql

7. 서비스 재시작 (matjip-db 제거 + matjip-backend 재연결)

   docker compose up -d --remove-orphans

8. 정상 동작 확인

   docker compose ps
   docker compose logs matjip-backend --tail=50

9. 이상 없으면 구 볼륨 정리 (선택)

   docker volume rm deploy_matjip_db_data


========================================
이후 코드 업데이트 반영 방법
========================================

# Statistics 업데이트
cd ~/Statistics && git pull origin main
cd ~/Statistics/deploy && docker compose up -d --build backend frontend

# Portfolio 업데이트
cd ~/Portfolio && git pull origin main
cd ~/Statistics/deploy && docker compose up -d --build portfolio-frontend

# MatJip 업데이트
cd ~/MatJip && git pull origin main
cd ~/Statistics/deploy && docker compose up -d --build matjip-backend matjip-frontend


========================================
관리 명령어
========================================

서비스 중지:       docker compose down
서비스 재시작:     docker compose restart
로그 확인:         docker compose logs -f [서비스명]
이미지 재빌드:     docker compose up -d --build [서비스명]

DB 접속:
   set -a; source .env; set +a
   # Statistics 스키마
   docker compose exec db mariadb -u$DB_USERNAME -p$DB_PASSWORD $DB_NAME
   # MatJip 스키마
   docker compose exec db mariadb -u$MATJIP_DB_USERNAME -p$MATJIP_DB_PASSWORD $MATJIP_DB_NAME
